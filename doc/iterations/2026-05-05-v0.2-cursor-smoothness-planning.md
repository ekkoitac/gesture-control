# Iteration Record: v0.2 Cursor Smoothness Planning

## Goal

- Open a new version focused on improving index-finger mouse-control smoothness.
- Keep this iteration limited to version planning docs and active TODO routing.

## Reason

- User validation found that current index-finger gesture cursor control is not smooth enough.
- The current v0.1 implementation maps the index fingertip to absolute screen coordinates with smoothing and a dead zone, which can still feel jumpy, laggy, or unstable depending on landmark noise and hand motion.

## Product Decisions

- Priority: make cursor control feel stable and responsive.
- Default direction: change cursor control from absolute screen positioning to relative movement.
- Safety gates remain unchanged: activation, pause, tracking loss, pinch, wave, and shortcut suppression should continue to win over cursor movement.

## Changes

- Added `doc/version/v0.2-cursor-smoothness/todo.md` as the new active version TODO.
- Updated version and agent entry docs so future work starts from `V2-01`.
- Added changelog coverage for the v0.2 planning start.

## Not Changed
