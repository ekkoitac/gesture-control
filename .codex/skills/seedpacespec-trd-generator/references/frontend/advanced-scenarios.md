# 客户端情况下,需求有多模版等场景的处理

> **加载条件**：≥5 个 R 或 ≥3 个架构层时读 §分块生成；PRD 涉及多模板时读 §多模板。

## 目标

复杂模块和多模板场景下，**不能一口气把整份 TRD 全吐完再让用户 review**。  
必须把 review 前移到生成过程中，用 **Codex 用户确认/输入流程** 做阶段确认，减少用户一次性审阅负担。

**这套“生成中确认”与主流程里的“整份 TRD 生成后确认”是并存关系，不是二选一：**

- **生成中确认**：用于复杂/多模板场景，边生成边确认方向
- **最终确认**：用于整份 TRD 写完后的总体验收

---

## 分块生成模式（Chunked Generation）

**适用**：涉及 5+ 个 R 或 3+ 个架构层的复杂模块。  
**不适用**：<5 R 且 ≤2 层的简单模块可直接全量生成。

### 强制规则

命中本模式后，**必须使用 向用户确认/请求输入 做生成中确认**。  
**禁止**直接整份生成完再让用户一次性 review。

### 流程

1. **先生成概要摘要**（§1 概述 + §2 矩阵 + §3 架构图示/数据流）
2. 用 **Codex 用户确认/输入流程** 询问用户是否认可总体方向
3. 用户确认后，再按块展开 §4
4. 每完成一大块，再次用 **Codex 用户确认/输入流程** 确认是否继续
5. 全部块确认后，再补 §5~§7
6. 最后仍进入主 skill 的**整份 TRD 最终确认循环**

### 用户确认最低要求

#### 第一次确认：概要方向

在输出概要摘要后，必须向用户确认/请求输入，类似：

> 已生成 TRD 概要摘要，是否认可这个整体方向？

选项至少包含：

- **A. 认可，继续展开**
- **B. 需要先调整概要**

#### 第二次及后续确认：分块展开

每完成一个大块后，必须向用户确认/请求输入，类似：

> 已完成 `{当前块}`，是否继续生成下一块？

选项至少包含：

- **A. 接受并继续**
- **B. 先修改这一块**
- **C. 后续全部直接继续，无需逐块确认**

### 推荐分块粒度

按下面粒度分块，避免一块过大：

1. **概要块**：§1 + §2 + §3
2. **核心方案块 1**：§4.1 ~ §4.2
3. **核心方案块 2**：§4.3 ~ §4.4
4. **核心方案块 3**：§4.5 ~ §4.7（若存在）
5. **收尾块**：§5 ~ §7

如果 UI 层特别重，也可以把 **§4.4 单独作为一个确认块**。

### 判断是否失败

下面任一情况，都算**未遵守本模式**：

- 命中复杂模块，但没有在生成过程中发起用户确认/输入流程
- 只在开头问一次，后续所有块直接生成
- 把整份 TRD 一次性生成完，再补一句“如需修改请提”
- 用户还没确认概要方向，就直接展开所有 §4

## 多模板场景处理

**适用**：纳入范围存在 ≥2 种模板/变体。两种触发路径：
- **多 R 模板组**：多个 R 属于同一功能域但各自代表不同模板（如 R-01 知识讲解、R-02 互动练习、R-03 总结回顾）
- **单 R 内多模板**：某个 R 的描述中列举了 ≥2 种呈现形态（如"知识讲解有 5 种模板"）

### 强制规则

命中多模板后，**模板本身也要分批确认**。  
不要一次把所有模板都展开完再让用户统一审。

### 结构要求

按以下结构组织 §4.4：

1. **§4.4.0 公共部分**：所有模板共享的组件/逻辑/状态
2. **§4.4.X 各模板**：每个模板特有的组件和逻辑
3. **模板实现指令表**：自然语言 ↔ 模板标识的对应

### 多模板确认流程

1. 先写 **§4.4.0 公共部分**
2. 用 **Codex 用户确认/输入流程** 确认公共部分是否认可
3. 再按模板逐个或逐组展开
4. 每完成一个模板（或一组模板），用 **Codex 用户确认/输入流程** 确认是否继续
5. 全部模板完成后，再进入整份 TRD 的最终确认

### 用户确认最低要求

#### 公共部分确认

> 已完成多模板公共部分，是否认可共享组件/状态/逻辑的设计？

选项至少包含：

- **A. 认可，继续模板细节**
- **B. 先修改公共部分**

#### 模板确认

> 已完成模板 `{模板名}`，是否继续生成下一个模板？

选项至少包含：

- **A. 接受并继续**
- **B. 先修改这个模板**
- **C. 后续模板无需逐个确认**

### 模板状态建议

每个模板在生成过程中可标记：

- ⬜ 待展开
- 🟡 已展开待用户确认
- ✅ 用户已确认

### 判断是否失败

下面任一情况，都算**未遵守多模板模式**：

- 公共部分和所有模板一次性全部输出
- 没有对公共部分单独确认
- 每个模板写完后没有 向用户确认/请求输入
- 用户还没确认前一个模板，就继续把后面模板全部写掉
